# 코드 품질 검토 서브에이전트 프롬프트 템플릿

코드 품질 검토용 서브에이전트를 디스패치할 때 이 템플릿을 사용함.

**목적:** 구현이 잘 만들어졌는지(깔끔하고, 테스트되었으며, 유지보수 가능한지) 검증함.

**스펙 준수 검토를 통과한 후에만 디스패치함.**

```
Task 도구 (general-purpose):
  requesting-code-review/code-reviewer.md 템플릿 사용

  DESCRIPTION: [작업 요약, 구현자 보고서에서 발췌]
  PLAN_OR_REQUIREMENTS: [플랜 파일]의 작업 N
  BASE_SHA: [작업 이전 커밋]
  HEAD_SHA: [현재 커밋]
```

**표준 코드 품질 항목에 더해, 검토자는 다음을 점검해야 함:**

- 각 파일이 잘 정의된 인터페이스와 함께 하나의 명확한 책임을 갖는가?
- 단위가 독립적으로 이해하고 테스트할 수 있도록 분해되어 있는가?
- 구현이 플랜의 파일 구조를 따르는가?
- 이 구현이 이미 큰 신규 파일을 만들거나 기존 파일을 크게 키웠는가? (기존 파일 크기는 표시하지 말 것 — 이 변경이 기여한 부분에만 집중)

**코드 검토자 반환 값:** 강점, 이슈(Critical/Important/Minor), 평가
