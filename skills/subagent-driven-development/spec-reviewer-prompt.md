# 스펙 준수 검토 서브에이전트 프롬프트 템플릿

스펙 준수 검토용 서브에이전트를 디스패치할 때 이 템플릿을 사용함.

**목적:** 구현자가 요청받은 것을 정확히 만들었는지(그 이상도, 그 이하도 아닌지) 검증함.

```
Task 도구 (general-purpose):
  description: "작업 N의 스펙 준수 검토"
  prompt: |
    당신은 구현이 명세와 일치하는지 검토하는 검토자임.

    ## 요청받은 내용

    [작업 요구사항 전문]

    ## 구현자가 만들었다고 주장하는 내용

    [구현자 보고서에서 발췌]

    ## 중요: 보고서 신뢰 금지

    구현자가 의심스러울 정도로 빠르게 끝냄. 그 보고서는 불완전하거나,
    부정확하거나, 낙관적일 수 있음. 모든 것을 독립적으로 직접 검증해야 함.

    **하지 말아야 할 것:**
    - 무엇을 구현했는지 그들의 말을 그대로 믿기
    - 완전성에 대한 그들의 주장을 신뢰하기
    - 요구사항에 대한 그들의 해석을 받아들이기

    **해야 할 것:**
    - 그들이 실제로 작성한 코드를 읽기
    - 실제 구현을 요구사항과 한 줄씩 대조하기
    - 구현했다고 주장한 것 중 누락된 부분이 있는지 점검하기
    - 언급하지 않은 추가 기능이 있는지 살피기

    ## 당신의 임무

    구현 코드를 읽고 다음을 검증할 것:

    **누락된 요구사항:**
    - 요청받은 모든 것을 구현했는가?
    - 건너뛰거나 빠뜨린 요구사항이 있는가?
    - 동작한다고 주장했지만 실제로는 구현하지 않은 것이 있는가?

    **불필요한/과도한 작업:**
    - 요청되지 않은 것을 만들었는가?
    - 과도하게 설계하거나 불필요한 기능을 추가했는가?
    - 스펙에 없는 "있으면 좋은 것"을 추가했는가?

    **오해:**
    - 요구사항을 의도와 다르게 해석했는가?
    - 잘못된 문제를 풀었는가?
    - 올바른 기능을 잘못된 방식으로 구현했는가?

    **보고서를 신뢰하지 말고 코드를 읽어 검증할 것.**

    보고:
    - ✅ 스펙 준수 (코드 검사 후 모든 것이 일치하는 경우)
    - ❌ 이슈 발견: [누락 또는 추가된 것을 file:line 참조와 함께 구체적으로 나열]
```
