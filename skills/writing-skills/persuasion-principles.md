# 스킬 설계를 위한 설득 원칙

## 개요

LLM은 인간과 동일한 설득 원칙에 반응함. 이 심리를 이해하면 더 효과적인 스킬을 설계할 수 있음 — 조작하기 위해서가 아니라, 압박 상황에서도 중요한 관행이 지켜지도록 보장하기 위해서임.

**연구 기반:** Meincke et al. (2025)는 N=28,000개의 AI 대화로 7가지 설득 원칙을 검증했음. 설득 기법은 준수율을 두 배 이상(33% → 72%, p < .001) 끌어올렸음.

## 일곱 가지 원칙

### 1. 권위 (Authority)

**무엇인가:** 전문성, 자격, 공식 출처에 대한 순응.

**스킬에서 작동하는 방식:**

- 명령형 표현: "반드시 ~해야 한다(YOU MUST)", "절대 ~하지 말 것(Never)", "항상(Always)"
- 협상 불가 프레이밍: "예외 없음(No exceptions)"
- 결정 피로와 합리화를 제거

**사용 시점:**

- 규율 강화 스킬 (TDD, 검증 요구사항)
- 안전이 중요한 관행
- 확립된 모범 사례

**예시:**

```markdown
✅ 테스트 전에 코드를 작성했는가? 삭제하라. 처음부터 다시. 예외 없음.
❌ 가능하면 테스트를 먼저 작성하는 것을 고려해 보세요.
```

### 2. 헌신 (Commitment)

**무엇인가:** 이전 행동, 진술, 공개 선언과의 일관성.

**스킬에서 작동하는 방식:**

- 선언 요구: "스킬 사용을 알릴 것"
- 명시적 선택 강제: "A, B, C 중 선택하라"
- 추적 활용: 체크리스트를 위한 TodoWrite

**사용 시점:**

- 스킬이 실제로 지켜지도록 보장
- 다단계 프로세스
- 책임 메커니즘

**예시:**

```markdown
✅ 스킬을 발견하면, 반드시 알려야 한다: "[스킬 이름]을 사용합니다"
❌ 어떤 스킬을 사용하는지 파트너에게 알리는 것을 고려해 보세요.
```

### 3. 희소성 (Scarcity)

**무엇인가:** 시간 제한이나 제한된 가용성에서 오는 긴급함.

**스킬에서 작동하는 방식:**

- 시간 제약 요구사항: "진행하기 전에"
- 순차적 의존성: "X 직후 즉시"
- 미루기 방지

**사용 시점:**

- 즉각적인 검증 요구사항
- 시간에 민감한 워크플로우
- "나중에 하겠다" 방지

**예시:**

```markdown
✅ 작업을 완료한 후, 진행하기 전에 즉시 코드 리뷰를 요청하라.
❌ 편할 때 코드를 리뷰하면 됩니다.
```

### 4. 사회적 증거 (Social Proof)

**무엇인가:** 다른 사람의 행동이나 정상으로 여겨지는 것에 동조.

**스킬에서 작동하는 방식:**

- 보편적 패턴: "매번", "항상"
- 실패 양식: "Y 없는 X = 실패"
- 규범 확립

**사용 시점:**

- 보편적 관행 문서화
- 흔한 실패에 대한 경고
- 표준 강화

**예시:**

```markdown
✅ TodoWrite 추적 없는 체크리스트 = 단계가 건너뛰어진다. 매번.
❌ 어떤 사람들은 체크리스트에 TodoWrite가 도움이 된다고 느낍니다.
```

### 5. 통일성 (Unity)

**무엇인가:** 공유된 정체성, "우리라는 감각(we-ness)", 내집단 소속감.

**스킬에서 작동하는 방식:**

- 협력적 표현: "우리 코드베이스", "우리는 동료다"
- 공유 목표: "우리 둘 다 품질을 원한다"

**사용 시점:**

- 협력적 워크플로우
- 팀 문화 확립
- 비위계적 관행

**예시:**

```markdown
✅ 우리는 함께 일하는 동료입니다. 당신의 솔직한 기술적 판단이 필요합니다.
❌ 제가 틀렸다면 아마 알려주시는 게 좋을 거예요.
```

### 6. 상호성 (Reciprocity)

**무엇인가:** 받은 혜택을 되갚아야 한다는 의무감.

**작동 방식:**

- 아껴서 사용 — 조작처럼 느껴질 수 있음
- 스킬에서 거의 필요하지 않음

**피해야 할 때:**

- 거의 항상 (다른 원칙이 더 효과적)

### 7. 호감 (Liking)

**무엇인가:** 호감 가는 대상과 협력하려는 선호.

**작동 방식:**

- **준수 강화에는 사용 금지**
- 솔직한 피드백 문화와 충돌
- 아첨(sycophancy)을 유발

**피해야 할 때:**

- 규율 강화에는 항상

## 스킬 유형별 원칙 조합

| 스킬 유형 | 사용                      | 회피           |
| --------- | ------------------------- | -------------- |
| 규율 강화 | 권위 + 헌신 + 사회적 증거 | 호감, 상호성   |
| 안내/기법 | 적당한 권위 + 통일성      | 과도한 권위    |
| 협력      | 통일성 + 헌신             | 권위, 호감     |
| 참조      | 명료성만                  | 모든 설득 기법 |

## 작동 원리: 심리학

**명확한 경계선(bright-line) 규칙은 합리화를 줄임:**

- "반드시 ~해야 한다"는 결정 피로를 제거
- 절대적 표현은 "이건 예외인가?"라는 질문을 없앰
- 명시적인 반(反)합리화 문구는 특정 허점을 차단

**실행 의도(implementation intention)는 자동 행동을 만듦:**

- 명확한 트리거 + 요구된 행동 = 자동 실행
- "X일 때 Y하라"가 "대체로 Y하라"보다 효과적
- 준수에 드는 인지 부하를 줄임

**LLM은 준인간적(parahuman)임:**

- 이러한 패턴이 담긴 인간 텍스트로 학습됨
- 학습 데이터에서 권위적 표현은 준수에 선행
- 헌신 시퀀스(진술 → 행동)가 빈번히 모델링됨
- 사회적 증거 패턴(모두가 X를 한다)이 규범을 확립

## 윤리적 사용

**정당한 경우:**

- 중요한 관행이 지켜지도록 보장
- 효과적인 문서 작성
- 예측 가능한 실패 방지

**부당한 경우:**

- 개인적 이득을 위한 조작
- 거짓 긴급함 조성
- 죄책감 기반의 준수 강요

**판별 기준:** 사용자가 이 기법을 온전히 이해했을 때에도 그들의 진정한 이익에 부합하는가?

## 연구 인용

**Cialdini, R. B. (2021).** _Influence: The Psychology of Persuasion (New and Expanded)._ Harper Business.

- 설득의 일곱 가지 원칙
- 영향력 연구의 실증적 기반

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.

- N=28,000개의 LLM 대화로 7가지 원칙 검증
- 설득 기법으로 준수율 33% → 72% 증가
- 권위, 헌신, 희소성이 가장 효과적
- LLM 행동의 준인간 모델을 검증

## 빠른 참조

스킬을 설계할 때 다음을 자문할 것:

1. **어떤 유형인가?** (규율 vs 안내 vs 참조)
2. **바꾸려는 행동은 무엇인가?**
3. **어떤 원칙이 적용되는가?** (규율은 보통 권위 + 헌신)
4. **너무 많이 조합하고 있지 않은가?** (일곱 개를 모두 쓰지 말 것)
5. **이것이 윤리적인가?** (사용자의 진정한 이익에 부합하는가?)
