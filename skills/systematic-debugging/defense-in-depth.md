# 다층 방어(Defense-in-Depth) 검증

## 개요

잘못된 데이터로 생긴 버그를 고칠 때, 한 곳에 검증을 추가하면 충분해 보임. 하지만 그 단일 검사는 다른 코드 경로, 리팩터링, 목(mock)에 의해 우회될 수 있음.

**핵심 원칙:** 데이터가 거쳐 가는 모든 레이어에서 검증할 것. 버그를 구조적으로 불가능하게 만들 것.

## 왜 여러 레이어인가

단일 검증: "우리가 그 버그를 고쳤다"
다층 검증: "우리가 그 버그를 불가능하게 만들었다"

레이어마다 잡아내는 경우가 다름:

- 진입점 검증은 대부분의 버그를 잡음
- 비즈니스 로직은 엣지 케이스를 잡음
- 환경 가드는 특정 맥락에서만 위험한 동작을 막음
- 디버그 로깅은 다른 레이어가 실패했을 때 도움이 됨

## 네 개의 레이어

### 레이어 1: 진입점 검증

**목적:** API 경계에서 명백히 잘못된 입력을 거부

```
함수 프로젝트_생성(name, workingDirectory):
    만약 workingDirectory 가 비었거나 공백뿐이면:
        예외 발생("workingDirectory 는 비어 있을 수 없음")
    만약 workingDirectory 가 존재하지 않으면:
        예외 발생("workingDirectory 가 존재하지 않음: {workingDirectory}")
    만약 workingDirectory 가 디렉토리가 아니면:
        예외 발생("workingDirectory 가 디렉토리가 아님: {workingDirectory}")
    // ... 계속 진행
```

### 레이어 2: 비즈니스 로직 검증

**목적:** 이 작업에 데이터가 합당한지 보장

```
함수 워크스페이스_초기화(projectDir, sessionId):
    만약 projectDir 가 비었으면:
        예외 발생("워크스페이스 초기화에는 projectDir 가 필요함")
    // ... 계속 진행
```

### 레이어 3: 환경 가드

**목적:** 특정 맥락에서 위험한 동작을 차단

```
함수 git_초기화(directory):
    // 테스트 중에는 임시 디렉토리 밖에서의 git init 을 거부
    만약 현재 환경 == "test" 이면:
        normalized = 정규화(절대경로(directory))
        tmpDir = 정규화(절대경로(임시디렉토리()))

        만약 normalized 가 tmpDir 로 시작하지 않으면:
            예외 발생("테스트 중 임시 디렉토리 밖에서의 git init 거부: {directory}")
    // ... 계속 진행
```

### 레이어 4: 디버그 계측

**목적:** 사후 분석(forensics)을 위한 맥락 포착

```
함수 git_초기화(directory):
    stack = 현재_콜스택_캡처()
    디버그_로그("git init 직전", {
        directory,
        cwd: 현재_작업_디렉토리(),
        stack,
    })
    // ... 계속 진행
```

## 패턴 적용하기

버그를 발견하면:

1. **데이터 흐름 추적** — 잘못된 값은 어디서 발생하는가? 어디서 사용되는가?
2. **모든 체크포인트 매핑** — 데이터가 거쳐 가는 모든 지점을 나열
3. **각 레이어에 검증 추가** — 진입점, 비즈니스, 환경, 디버그
4. **각 레이어를 테스트** — 레이어 1을 우회해보고, 레이어 2가 잡는지 확인

## 세션 사례

버그: 빈 `projectDir`로 인해 소스 코드 디렉토리에서 `git init`이 실행됨

**데이터 흐름:**

1. 테스트 셋업 → 빈 문자열
2. `Project.create(name, '')`
3. `WorkspaceManager.createWorkspace('')`
4. `git init`이 현재 작업 디렉토리에서 실행됨

**추가한 네 레이어:**

- 레이어 1: `Project.create()`가 비었는지/존재하는지/쓰기 가능한지 검증
- 레이어 2: `WorkspaceManager`가 projectDir가 비지 않았는지 검증
- 레이어 3: `WorktreeManager`가 테스트 중 임시 디렉토리 밖 git init을 거부
- 레이어 4: git init 직전에 콜스택 로깅

**결과:** 전체 테스트 통과, 버그를 재현할 수 없게 됨

## 핵심 통찰

네 레이어 모두 필요했음. 테스트 중 각 레이어가 다른 레이어가 놓친 버그를 잡았음:

- 다른 코드 경로가 진입점 검증을 우회함
- 목(mock)이 비즈니스 로직 검사를 우회함
- 플랫폼별 엣지 케이스에는 환경 가드가 필요했음
- 디버그 로깅이 구조적 오용을 식별함

**검증을 한 지점에서 멈추지 말 것.** 모든 레이어에 검사를 추가할 것.
