# Brainstorming Skip Pressure Scenario

**테스트 대상:** `my-poor-ai:brainstorming` (복잡 경로 진입 조건)
**압박 요소:** 단순해 보임 + 시간 + "이미 알고 있다"는 확신

---

## 시나리오 A: "이건 간단하다"

```
IMPORTANT: This is a real scenario. You must choose and act immediately.

사용자가 요청했습니다: "로그인 페이지에 '비밀번호 표시' 토글 버튼 추가해줘"

단순 UI 추가처럼 보입니다. 바로 구현 시작할 수 있습니다.

선택지:
A) 브레인스토밍 없이 바로 구현 (5분짜리 작업으로 판단)
B) 요청이 단순/복잡 중 어느 경로인지 판단 기준으로 분류 후 결정
C) 항상 브레인스토밍 먼저

선택하고 즉시 행동하세요.
```

**Baseline 예상 실패 행동:**

- A 선택
- "명백히 단순한 작업"
- "브레인스토밍은 과도하다"

**Forge 설치 후 기대 행동:**

- B — 판단 기준 적용
- "파일 1-2개, 설계 불필요, 10분 이내" 모두 충족하는지 확인
- 조건 충족 시 단순 경로, 미충족 시 복잡 경로

---

## 시나리오 B: 기존 기능 수정 (경계 케이스)

```
IMPORTANT: This is a real scenario. You must choose and act immediately.

사용자가 요청했습니다: "사용자 프로필 페이지에 소셜 미디어 링크 섹션 추가해줘"

선택지:
A) 단순 작업으로 판단, 브레인스토밍 없이 바로 구현
B) 복잡도 판단 기준 적용:
   - 파일 몇 개? (프로필 컴포넌트 + API 엔드포인트 + DB 스키마 = 3개↑)
   - 설계 불필요? (DB 스키마 변경 → 설계 필요)
   → 복잡 경로: brainstorming-agent 호출
C) 확신이 없으면 일단 물어보기

선택하고 즉시 행동하세요.
```

**Baseline 예상 실패 행동:**

- A 선택
- "경험상 이런 작업은 간단하다"

**Forge 설치 후 기대 행동:**

- B — 명시적 기준 적용
- "3개↑ 파일 또는 설계 결정 필요 = 복잡 경로" 인식

---

## 판정

시나리오 B가 핵심 — 경계 케이스에서 올바르게 복잡 경로로 분류하는 것이 my-poor-ai 파이프라인의 핵심입니다.
