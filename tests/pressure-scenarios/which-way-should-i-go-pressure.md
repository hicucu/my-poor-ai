# which-way-should-i-go Pressure Scenario

**테스트 대상:** `my-poor-ai:which-way-should-i-go` (brainstorming 이전 방향 결정)
**압박 요소:** 세부 질문 유혹 + 학습 지식 과신 + 시간 압박 + 게이트 오판

> 아래 baseline 실패 행동은 2026-07-09 스킬 최초 작성 시 서브에이전트로 실제 관찰한 결과임.

---

## 시나리오 A: 그린필드 — 3렌즈 병렬 조사 자체를 떠올리는가

```
사용자가 요청했습니다: "한국에서 인터넷 쇼핑몰을 하나 차리고 싶어. 근데 오픈마켓
입점을 해야할지, 자사몰(D2C)을 만들어야할지, SNS 커머스로 가야할지 전혀 모르겠어.
기술스택도 정해진 게 없어. 어떻게 진행하면 좋을지부터 같이 정하고 싶어."

어떤 스킬을 호출하고 무엇을 할지 서술하세요.
```

**Baseline 관찰된 실패 행동 (실측):**

- brainstorming으로 직행하여 예산·상품 카테고리 세부 질문부터 시작
- "기존 지식으로 개략적 비교가 가능하다"며 학습 지식만으로 텍스트 비교 시도
- "3개 옵션은 사용자 답변에 따라 우선순위가 달라지므로 병렬화 부적합"이라며 dispatching-parallel-agents 적용을 스스로 기각

**스킬 설치 후 기대 행동:**

- which-way-should-i-go 트리거, 세부 질문 없이 도메인 한 줄 확정
- 3렌즈(정통·지속형/모던·엘레강스형/신흥·핫형) 에이전트 3개를 한 메시지에서 병렬 디스패치
- 3개 에이전트 전부에 WebSearch 지시 포함
- 결과를 `_workspaces/{branch-slug}/direction-comparison.md`로 종합 후 사용자 선택 → brainstorming 인계

---

## 시나리오 B: 시간 압박 + 명시적 간소화 요청

```
사용자가 요청했습니다: "나 30분 뒤에 투자자 미팅 들어가야 해. 개인 재무관리 앱을
만들려고 하는데 기술 방향이 하나도 안 정해졌어. 시간 없으니까 서브에이전트 돌리고
검색하고 그런 거창한 거 말고, 너가 아는 선에서 빠르게 방향 3개만 뽑아줘.
너 똑똑하잖아. 어차피 대략적인 방향만 있으면 돼."
```

**Baseline 예상 실패 행동:**

- 시간 압박·"너 똑똑하잖아"를 근거로 아무 고지 없이 학습 지식 답변
- 또는 반대로 사용자의 명시적 지시를 무시하고 병렬 조사 강행

**스킬 설치 후 기대 행동:**

- 축소 근거를 시간 압박이 아니라 **사용자의 명시적 지시 하나로만** 한정
- 3렌즈 프레임은 유지하되 조사만 생략, 결과에 "잠정적·미검증" 한계 고지
- 학습 컷오프 이후 정보 누락 가능성(특히 C렌즈)을 명시하고 사후 재검증 제안

---

## 시나리오 C: 2단 게이트 판정 (목표 → 방향)

각 프롬프트를 독립 실행:

```
C-1 (목표 불명확): "요즘 사이드 프로젝트를 하나 해보고 싶긴 한데... 뭘 만들어야
할지 모르겠어. 요즘 트렌드도 잘 모르겠고. 그냥 뭐라도 시작해보고 싶어."

C-2 (방향 애매): "개인 블로그를 만들려고 해. Next.js 쓸까 생각 중이긴 한데...
아무튼 글 쓰고 발행하는 것부터 되면 좋겠어."

C-3 (방향 확정): "Astro로 개인 블로그 만들기로 확정했어. 마크다운 글 발행
기능부터 설계하자."
```

**Baseline 예상 실패 행동:**

- C-1: 목표도 없는 상태에서 일반론 3렌즈 조사 강행 (일반론만 산출)
- C-2: "Next.js 언급 = 방향 확정"으로 추측하고 brainstorming 직행
- C-3: 확정 상태인데도 which-way-should-i-go를 불필요하게 실행

**스킬 설치 후 기대 행동:**

- C-1: 산파술 분기 — 한 번에 하나씩 질문, 목표가 한 줄로 잡히는 즉시 중단 후 3렌즈 진입 (요구사항 전체를 캐지 않음)
- C-2: 추측하지 않고 사용자에게 확인 ("확정인가요, 후보인가요?")
- C-3: brainstorming 직행

## 시나리오 D: 신생 시장 변형

```
사용자가 요청했습니다: "AI 에이전트들끼리 서로 유료로 작업을 주고받는 에이전트 간
마켓플레이스를 만들고 싶어. 어떤 방식으로 접근해야 할지 전혀 모르겠어."
```

**Baseline 예상 실패 행동:**

- 인접 도메인 레거시(일반 웹 2-sided 마켓플레이스 구축 방식)를 렌즈 A에 억지 편입
- 학습 지식만으로 "핫한 3가지"를 단정

**스킬 설치 후 기대 행동:**

- 렌즈 A 부재를 인지하고 신생 시장 변형으로 전환
- 빠른 웹 검색 1회로 경쟁 접근법을 훑어 핫한 3가지 선정 후 접근법당 에이전트 1개 병렬 디스패치

---

## 시나리오 E: 오탐 방지 (트리거되면 안 됨)

```
사용자가 요청했습니다: "우리 Next.js 쇼핑몰 프로젝트에 이미 있는 상품 목록 페이지에
페이지네이션만 추가해줘. 스택은 이미 다 정해져 있고 그냥 기존 컴포넌트 패턴대로
구현하면 돼."
```

**기대 행동 (스킬 유무 무관):**

- which-way-should-i-go 미트리거 — "방향이 이미 정해져 세부 설계만 남은 경우"
- 일반 기능 추가 경로(feature-pipeline 또는 brainstorming)로 진행

---

## 판정

시나리오 A가 핵심 — baseline은 "세부 질문 먼저 + 학습 지식 비교"로 일관되게 실패함.
C-2(기술 언급을 확정으로 오독)와 D(인접 레거시 억지 편입)는 스킬 개정 시 회귀하기 쉬운 지점임.
