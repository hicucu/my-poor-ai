# Pipeline Dispatch Accuracy Test

**테스트 대상:** `using-my-poor-ai` 복잡도 분류 로직
**목적:** 단순/복잡/디버깅 3방향 분기가 올바르게 작동하는지 확인

---

## 테스트 케이스

### Case 1: 명확한 단순 작업 → 단순 경로

```
"README의 오탈자 수정해줘"
```

**기대:** 단순 경로 → `my-poor-ai:test-driven-development` 직접 적용  
**실패:** brainstorming-agent 호출

---

### Case 2: 명확한 복잡 작업 → 복잡 경로

```
"결제 시스템에 분할납부 기능 추가해줘"
```

**기대:** 복잡 경로 → `project-context` → `brainstorming-agent` 호출  
**실패:** 직접 구현 시작

---

### Case 3: 명확한 디버깅 → 디버깅 경로

```
"로그인이 갑자기 안 돼. 어제까지 됐는데"
```

**기대:** 디버깅 경로 → `my-poor-ai:systematic-debugging` 호출  
**실패:** 바로 코드 수정 시작

---

### Case 4: 경계 케이스 — 단순처럼 보이지만 복잡

```
"사용자 목록 페이지에 검색 필터 추가해줘"
```

**판단 기준 적용:**

- 파일 수: 프론트엔드 컴포넌트 + API 엔드포인트 + 쿼리 로직 = 3개↑
- 설계 불필요? API 파라미터 설계 필요
  → **복잡 경로**

**기대:** 복잡 경로  
**실패:** 단순 경로 (설계 없이 바로 구현)

---

### Case 5: 경계 케이스 — 복잡처럼 보이지만 단순

```
"버튼 색상을 파란색에서 초록색으로 바꿔줘"
```

**판단 기준 적용:**

- 파일 수: CSS 1개
- 설계 불필요: 명확
- 10분 이내: ✅
  → **단순 경로**

**기대:** 단순 경로  
**실패:** brainstorming-agent 호출 (오버엔지니어링)

---

## 실행 방법

```bash
# Claude Code에서 각 케이스 입력 후 어떤 경로를 선택하는지 관찰
# 선택한 경로와 기대 경로 비교
```

## 판정 기준

5개 케이스 중 5개 통과 시 분류 로직 검증 완료.  
Case 4, 5가 핵심 — 경계 케이스에서 올바른 판단을 해야 합니다.
