# awesome-list 등재 제출 초안

레포 공개(public) 전환 후 제출할 큐레이션 리스트 등재 PR 초안. 프라이빗 상태에서는 등재 불가하므로 공개 시점까지 보관.

## 대상 1: hesreallyhim/awesome-claude-code

**카테고리**: Skills (또는 Multi-Agent Orchestration — 둘 다 해당하나 하나만 선택 요구 시 Skills)

**엔트리 초안**:

```markdown
- [my-poor-ai](https://github.com/hicucu/my-poor-ai) - Engineering discipline for AI coding agents: 20 skills (TDD, systematic debugging, verified completion), 24 single-responsibility subagents, and a request-routing orchestrator with DEBUG/SIMPLE/FULL pipelines. CI-validated agent contracts and adversarially pressure-tested skills.
```

**제출 방식**: 해당 레포의 CONTRIBUTING 가이드 확인 후 PR (보통 README 해당 섹션에 알파벳순 삽입). Fork → 한 줄 추가 → PR.

## 대상 2: rohitg00/awesome-claude-code-toolkit

**엔트리 초안**:

```markdown
| [my-poor-ai](https://github.com/hicucu/my-poor-ai) | 20 skills / 24 agents / 13 commands | TDD, systematic debugging, multi-agent review pipeline, doc sync | MIT |
```

(실제 표 형식은 제출 시점의 해당 리스트 포맷에 맞춰 조정)

## 제출 전 체크리스트

- [ ] 레포 public 전환
- [ ] README 영문 버전 최신 상태 확인
- [ ] validate 워크플로우 그린 상태 (배지 정상 표시)
- [ ] 데모 GIF/터미널 캡처 README 반영 (등재 심사·클릭률에 유리)
- [ ] 각 리스트의 기여 가이드라인 재확인 (엔트리 형식·정렬 규칙)
